# B站上传流程（按 2026-04-27 录屏还原）

来源录屏：
`/Users/a001/Desktop/录屏/录屏2026-04-27 10.02.34.mov`

目标：
- 一条视频一个稿件
- 不改标题/分区/标签也能直接投
- 上传完成后立刻投稿，再进入下一条

## 录屏里的稳定流程

1. 在 B站创作中心视频投稿页点击 `上传视频`。
2. 系统文件选择器打开后，直接进入真实素材目录。
3. 单击目标视频文件，让右下角 `上传` 按钮变蓝。
4. 点击系统文件选择器右下角 `上传`。
5. 等待 B站进入稿件编辑页，顶部出现视频卡片与上传进度。
6. 不填额外字段，直接滚到页面底部。
7. 点击 `立即投稿`。
8. 等到 `稿件投递成功` 页面出现。
9. 点击 `再投一个`，回到上传页继续下一条。

## 录屏关键节点

- `00s`：已经在视频投稿页，准备点 `上传视频`
- `03s`：系统文件选择器已打开，并停在真实作品目录
- `15s`：视频已进入编辑页，上传卡片出现
- `24s`：页面已滚到底部，`立即投稿` 可点
- `30s`：出现 `稿件投递成功`
- `42s`：开始下一轮，再次打开系统文件选择器
- `45s`：第二条视频已进入编辑页
- `54s`：第二条再次点 `立即投稿`
- `57s`：再次成功

## 自动化约束

这段流程最关键的不是网页按钮，而是“系统文件选择器里必须选中真实文件”：

- 可以用真实文件
- 可以用硬链接
- 可以用 APFS clone / 真复制
- 不要用软链接 `symlink`

原因：
- 软链接在 Safari 的文件选择器里容易被当成别名或跳回原目录
- 表面上看像选中了，实际上网页上传控件没有真正吃到文件

## 建议的自动化拆法

### A. 中转准备

先把当前要传的视频做成一个干净中转目录：

- 目录：`~/Downloads/sivan-bili-stage`
- 文件名：统一短 ASCII 名
- 映射表：`_mapping.tsv`

当前已经有脚本：
[`prepare_bilibili_stage.py`](/Users/a001/Documents/工作树/升级优化类/个人进化类/animation-sequence-123/scripts/prepare_bilibili_stage.py)

它默认会：
- 读取当前 `scripts/bilibili_upload_manifest.json`
- 把视频做成真实中转文件
- 优先用硬链接，失败再复制
- 生成 `_mapping.tsv`

### B. 浏览器动作

浏览器自动化只做这几件事：

1. 点上传页 `上传视频`
2. 在文件选择器中选中中转目录里的下一条文件
3. 点文件选择器的 `上传`
4. 等到编辑页出现视频卡片
5. 滚到底
6. 点 `立即投稿`
7. 等待成功页
8. 点 `再投一个`

### C. 失败判定

如果以下任一条件不满足，就不能继续下一条：

- 编辑页顶部没有出现新视频卡片
- 页面标题仍停在空投稿页
- 成功页没有出现 `稿件投递成功`

## 推荐命令

先准备一批真实中转文件：

```bash
python3 scripts/prepare_bilibili_stage.py
```

如果只想先准备 10 条：

```bash
python3 scripts/prepare_bilibili_stage.py --limit 10
```

如果有已上传名单文件：

```bash
python3 scripts/prepare_bilibili_stage.py --exclude-file uploaded_paths.txt
```
